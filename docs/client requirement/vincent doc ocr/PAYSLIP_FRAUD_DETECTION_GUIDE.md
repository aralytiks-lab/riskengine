# Swiss Payslip Fraud Detection Guide

## Overview

This guide documents fraud detection patterns identified through analysis of Swiss payslips submitted to LeaseTeq AG. It provides a systematic approach to identifying fraudulent documents.

---

## 1. Metadata Analysis

### Critical Metadata Fields to Check

| Field | What to Look For | Red Flag |
|-------|------------------|----------|
| **Author/Creator** | Should be payroll software (Abacus, Sage, SAP, Simultan) | Personal usernames (e.g., "profcan"), "Word", "Pages" |
| **Producer** | Payroll system or legitimate PDF generator | Microsoft Word, iPhone/iOS, Canva |
| **Creation Date** | Should align with payslip period | Created recently for old payslips |
| **Title** | Should match document content | Title differs from content (e.g., "April 24" but content shows "November 2025") |

### How to Extract Metadata

**macOS Terminal:**
```bash
# Local file metadata
mdls "/path/to/payslip.pdf"

# Internal PDF metadata
strings "/path/to/payslip.pdf" | grep -iE "(CreationDate|ModDate|Producer|Creator|Author|Title)"
```

### Metadata Red Flags by Creator

| Creator | Risk Level | Notes |
|---------|------------|-------|
| Microsoft Word | HIGH | Payslips should never be created in Word |
| iOS Quartz PDFContext | HIGH | Created on iPhone - not payroll software |
| Canva | HIGH | Design tool, not payroll system |
| macOS Quartz PDFContext | LOW-MEDIUM | Could be legitimate scan |
| Abacus, Sage, SAP | LOW | Legitimate payroll systems |

## 2. SwissDec Verification Standards

### What is SwissDec?

SwissDec is the Swiss standard for electronic salary data exchange (Einheitliches Lohnmeldeverfahren / ELM). The SwissDec association certifies payroll software that meets legal requirements for transmitting salary data to:
- AHV/AVS (old-age insurance)
- Accident insurance (SUVA, UVG)
- Tax authorities (Quellensteuer)
- Federal Statistical Office
- Pension funds

**Key principle:** Legitimate Swiss payslips come from SwissDec-certified payroll software, not Microsoft Word or design tools.

### Fraud Detection: Metadata vs. Claimed Software

**CRITICAL CHECK:** If a payslip appears to come from a certified system but metadata shows otherwise → FRAUDULENT

| Payslip Claims | Metadata Shows | Verdict |
|----------------|----------------|---------|
| "Generated by Abacus" | Creator: Microsoft Word | **FRAUDULENT** |
| Professional Lohnart layout | Author: personal email | **FRAUDULENT** |
| Company uses SAP | Producer: iPhone iOS | **FRAUDULENT** |
| Looks like bexio template | Created in Canva | **FRAUDULENT** |

**Why this matters:** Fraudsters often copy the visual layout of legitimate payslips but create them in Word. The metadata exposes this.

### Lohnart Code Structure (Wage Type Codes)

Professional SwissDec-certified payroll systems use standardized 4-digit Lohnart codes organized by category:

#### Standard Numbering Convention

| Code Range | Category | Examples |
|------------|----------|----------|
| **1000-1999** | Income / Bruttolohn | 1001 Monatslohn, 1100 Stundenlohn, 1405 13. Monatslohn |
| **2000-2999** | Deductions / Abzüge | 2001 AHV, 2005 ALV, 2101 KTG, 2501 BVG |
| **3000-3999** | Employer contributions | 3001 AHV-AG, 3005 ALV-AG, 3101 BU-Prämie |
| **5000+** | Net calculations / Totals | 5001 Nettolohn, 5999 Auszahlung |
| **6000-6999** | Information / Tracking | 6501 Feriensaldo, 6505 Ferienbezug |

**Key rule:** Codes under 5000 contribute to gross salary calculation.

#### Fraud Detection via Lohnart Codes

| Indicator | Legitimate | Fraudulent |
|-----------|------------|------------|
| Code format | 4-digit numeric (1001, 2005) | No codes, or random text |
| Code grouping | Logical sequence (2001→2005→2101) | Scattered or missing |
| Income codes | 1xxx series | Just "Bruttolohn" with no code |
| Deduction codes | 2xxx series | Listed without codes |
| Employer contributions | 3xxx series visible | Never shown |
| Sub-numbers | Present for automatic deductions | Absent |

**Example - Legitimate (Carrosserie Hess AG):**
```
1001  Salär                    7,025.00
1405  13. Gehalt               7,025.00
1999  Bruttolohn              14,050.00
2001  AHV-Abzug     5.30%       -744.65
2005  ALV-Abzug     1.10%       -154.55
2101  KTG-Abzug     0.81%       -113.80
2501  PK-Beitrag                -247.95
```

**Example - Fraudulent (typical Word-created):**
```
Bruttolohn                    6,300.00
AHV                            -333.90
ALV                             -69.30
Nettolohn                     5,306.50
```
(No Lohnart codes = not from payroll software)

### ELM Version Requirements (2026)

| Fiscal Year | Requirement |
|-------------|-------------|
| 2025 wages | ELM 4.0 accepted until March 2026 |
| 2026 wages (Jan 1+) | **ELM 5.0 mandatory** |

**Implication:** Any employer claiming to use certified software in 2026 should be on ELM 5.0-capable systems.

### SwissDec Verification Checklist

```
1. Extract PDF metadata (Creator, Author, Producer)
2. Compare against certified software list
   ├─ Match? → Proceed to content verification
   └─ Mismatch (Word/iPhone/personal name)? → HIGH RISK
3. Check for Lohnart codes
   ├─ Present with logical 4-digit format? → Legitimate indicator
   └─ Absent or random format? → SUSPICIOUS
4. Verify code categories
   ├─ 1xxx for income, 2xxx for deductions? → Good
   └─ No systematic structure? → SUSPICIOUS
5. Cross-reference employer
   └─ Large company claims manual payslip? → FRAUDULENT
```

## 4. Swiss Social Contribution Rates (2025)

### Mandatory Rates

| Contribution | Employee Rate | Notes |
|--------------|---------------|-------|
| AHV/AVS/AI/APG | **5.30%** | Fixed rate |
| ALV/AC (Unemployment) | **1.10%** | Up to CHF 148,200/year |
| ALV2 (Solidarity) | 0.50% | Only on income > CHF 148,200 |

### Variable Rates

| Contribution | Typical Range | Notes |
|--------------|---------------|-------|
| BVG/LPP/PK (Pension) | 7-18% | Age-dependent |
| NBU/LAA/ANP (Accident) | 0.5-3% | Industry-dependent |
| KTG/IJM (Sickness) | 0.5-2% | Policy-dependent |

### Industry-Specific

| Industry | Special Requirement |
|----------|---------------------|
| Gastronomy (Gastro) | L-GAV contribution mandatory (~CHF 20/month) |
| Construction | FAR/Parifonds contributions |
| Retail | Possible GAV contributions |

## 5. Quellensteuer (Source Tax)

### Valid Tariff Codes

| Code | Description |
|------|-------------|
| A | Single, no children |
| B | Married, sole earner |
| C | Married, dual income |
| H | Single parent |
| D, E, F | Special cases |

### Invalid Codes
- **"L"** - Does not exist (seen in fraudulent bate.pdf)
- Any other letters not in standard list

### When Required
- B-permit holders (Aufenthaltsbewilligung B)
- Cross-border workers (Grenzgänger) in some cases
- Swiss/C-permit holders are generally exempt

## 6. Mathematical Verification Checklist

### Step-by-Step Verification

```
1. Verify AHV: Gross × 5.30% = Stated AHV
2. Verify ALV: Gross × 1.10% = Stated ALV
3. Verify other deductions at stated rates
4. Sum all deductions = Total Abzüge
5. Gross - Total Abzüge = Nettolohn
6. Nettolohn - other deductions = Auszahlung
```

### Swiss Rounding Rules
- All amounts should round to nearest CHF 0.05 (5 Rappen)
- Example: 127.26 should be 127.25 or 127.30

## 7. Document Authenticity Indicators

### Positive Indicators (Legitimate)

| Indicator | Why It Matters |
|-----------|----------------|
| Lohnart codes (1001, 2001, etc.) | Professional payroll software |
| Employee ID (MA-Nr, Pers-Nr) | Systematic HR tracking |
| Cost center (Kostenstelle) | Internal accounting system |
| Vacation tracking | Complex data rarely faked |
| 13th salary in December | Swiss standard practice |
| Company-specific deductions | Authentic operational detail |
| Valutadatum (payment date) | Specific date within period |

### Negative Indicators (Fraudulent)

| Indicator | Why It's Suspicious |
|-----------|---------------------|
| Created in Word/iPhone | Not payroll software |
| Personal author name | Individual, not system |
| Metadata title mismatch | Template reuse |
| Missing mandatory contributions | Unfamiliarity with Swiss law |
| Invalid tax codes | Fabricated data |
| Round salary amounts only | Real salaries have centimes |
| No year on payslip | Basic error |

**Note:** Identical payslips across multiple months is NOT a fraud indicator. Fixed-salary employees (Monatslohn) with no overtime, bonuses, or absences legitimately have identical payslips month-to-month.

## 9. Quick Reference: Fraud Detection Flowchart

```
START
  │
  ├─► 1. Check Metadata
  │     ├─ Author = "Word" or personal name? ──► HIGH RISK
  │     ├─ Created on iPhone? ──► HIGH RISK
  │     └─ Title ≠ Content? ──► FRAUDULENT
  │
  ├─► 2. SwissDec Verification
  │     ├─ Metadata shows Word but layout looks professional? ──► FRAUDULENT
  │     ├─ Creator NOT in certified software list? ──► SUSPICIOUS
  │     ├─ Lohnart codes present (1001, 2001, etc.)? ──► Legitimate indicator
  │     └─ No Lohnart codes at all? ──► SUSPICIOUS
  │
  ├─► 3. Check Required Elements
  │     ├─ Missing UID? ──► SUSPICIOUS
  │     ├─ Missing AHV? ──► SUSPICIOUS
  │     ├─ Missing BVG (salary > 22K)? ──► HIGH RISK
  │     └─ Missing year? ──► HIGH RISK
  │
  ├─► 4. Verify Mathematics
  │     ├─ AHV ≠ 5.30%? ──► SUSPICIOUS
  │     ├─ ALV ≠ 1.10%? ──► SUSPICIOUS
  │     └─ Totals don't match? ──► FRAUDULENT
  │
  ├─► 5. Check Consistency
  │     ├─ Invalid tax codes? ──► FRAUDULENT
  │     └─ Dates from distant past? ──► FRAUDULENT
  │
  └─► 6. Verify Company
        ├─ Company exists? ──► Check UID registry
        ├─ Large company but manual payslip? ──► FRAUDULENT
        └─ Address verifiable? ──► Cross-reference
```

## 10. Verification Resources

### Swiss Company Registry
- Zefix: https://www.zefix.ch
- Search by company name or UID

### Address Verification
- Swiss Post: https://www.post.ch/address-check
- Google Maps for location verification

### AHV Number Validation
- Format: 756.XXXX.XXXX.XX
- First 3 digits always 756 (Switzerland)
- Last 2 digits are checksum

### IBAN Validation
- Swiss IBANs: CH## #### #### #### #### #
- Bank clearing numbers:
  - 00xx = PostFinance
  - 002x = UBS
  - 00230-00239 = UBS
  - 80xxx = Raiffeisen
  - 00700-00799 = ZKB
